## EXPLAIN

**주제: 만든 session-wrap 스킬 실행 + 검증**

### 1. 스킬이 만들어졌으면 실행해봐야 한다

Block 1에서 `my-session-wrap` 스킬을 만들었습니다. 이제 실제로 실행해볼 차례입니다.

레시피를 작성했으면 요리를 해봐야 맛을 알 수 있습니다. 스킬도 마찬가지로, 만들기만 하면 의미가 없습니다. 직접 돌려보고 결과를 확인해야 진짜 내 것이 됩니다.

### 2. 실행 전 확인할 것

스킬을 실행하기 전에 3가지를 확인합니다.

- **파일 존재 여부**: `.claude/skills/my-session-wrap/SKILL.md` 파일이 실제로 있는지 확인합니다. 이 파일이 없으면 Claude는 스킬을 인식하지 못합니다.
- **frontmatter(이름표) 확인**: 파일 맨 위의 `name`과 `description`이 올바르게 적혀 있는지 봅니다. `name`은 스킬을 부르는 이름이 되고, `description`은 Claude가 "이 스킬을 언제 쓸지" 판단하는 기준이 됩니다.
- **Claude 인식 확인**: "어떤 스킬이 설치되어 있어?"라고 물어보면, Claude가 인식하고 있는 스킬 목록을 보여줍니다.

### 3. 실행 후 볼 것

스킬이 실행되면 아래 3가지가 순서대로 일어나야 합니다.

- **4개 에이전트 병렬 실행**: 화면에 "Task 실행 중..." 또는 에이전트 이름이 여러 개 표시되면 정상입니다. 4개 에이전트가 동시에 분석을 시작합니다.
- **Phase 2 검증 동작**: 에이전트들의 결과물을 종합하는 검증 단계가 실행됩니다. 이 과정에서 중복이 정리됩니다.
- **AskUserQuestion 선택지 제시**: 최종 결과를 바탕으로 "어떻게 할까요?"라는 선택지가 나옵니다. 사용자가 직접 다음 행동을 고르는 것입니다.

### 4. 문제가 있으면 수정

실행했는데 의도대로 동작하지 않을 수 있습니다. 그때는 아래를 확인합니다.

- **에이전트가 실행되지 않으면**: SKILL.md 안에 Task 호출 부분이 올바르게 작성되어 있는지 Claude에게 확인을 요청합니다.
- **병렬이 아니라 순차로 실행되면**: 하나의 메시지 안에 4개 Task 호출이 모두 들어 있는지 확인합니다. 따로따로 호출하면 순차 실행이 됩니다.
- **결과가 이상하면**: 각 에이전트에게 주는 지시문을 수정합니다. 지시가 모호하면 결과도 모호해집니다.

## EXECUTE

아래 순서대로 실행해보세요.

### Step 1. 새 터미널 탭에서 작업 세션 열기

스킬을 테스트하려면 **별도의 세션**이 필요합니다. 지금 이 교육 세션에서 직접 실행하면 교육 흐름이 끊길 수 있기 때문입니다.

**새 터미널 탭을 열어주세요:**
- Mac: `Cmd + T` (터미널에서 새 탭)
- 새 탭에서 같은 프로젝트 폴더로 이동합니다. 캠프 시작할 때 사용한 `cd` 명령어를 다시 입력하면 됩니다.
- 폴더로 이동한 뒤 `claude`를 입력해서 Claude Code를 시작합니다

> 이 교육 세션은 그대로 두고, 새 탭에서 작업합니다. Block 2 실습이 끝나면 이 교육 세션 탭으로 돌아오면 됩니다.

> **이 새 탭에서 그대로 Step 2~5를 진행합니다.** Step 1 확인 후 교육 탭으로 돌아가지 마세요.

새 세션에서 아래를 입력합니다:

```
어떤 스킬이 설치되어 있어?
```

Claude가 보여주는 목록에 `my-session-wrap`이 있는지 확인합니다. 목록에 없다면 `.claude/skills/my-session-wrap/SKILL.md` 파일이 올바른 위치에 있는지 다시 확인하세요.

### Step 2. 먼저 작업을 좀 하기

wrap(세션 정리)을 하려면 정리할 작업이 있어야 합니다. 간단한 작업을 먼저 해봅니다.

```
간단한 README.md 파일을 만들어줘. 내 이름, 오늘 날짜, "Day 4 실습"이라고 적어줘.
```

이렇게 하면 세션에 변경사항이 생기고, wrap 스킬이 분석할 대상이 만들어집니다.

### Step 3. 스킬 실행하기

아래 명령으로 만든 스킬을 실행합니다. Step 1에서 확인한 스킬 이름을 사용하세요:

```
/my-session-wrap
```

> 만약 "스킬을 찾을 수 없습니다"라고 나오면, Step 1에서 확인한 스킬 목록의 이름을 확인하세요. `name`이 다르게 설정되었을 수 있습니다.

자연어로도 실행할 수 있습니다:

```
세션 정리해줘
```

두 방법 모두 같은 스킬이 실행됩니다. `/` 명령어는 직접 호출, 자연어는 `description`을 기반으로 Claude가 자동으로 매칭합니다.

### Step 4. 결과 관찰하기

실행 결과를 주의 깊게 관찰합니다. 아래 질문에 답해보세요.

- 에이전트 4개가 실행되는 것이 보이는가? (화면에 "Task 실행 중..." 또는 에이전트 이름이 여러 개 표시되면 정상)
- 각 에이전트의 분석 결과가 나오는가?
- 최종 선택지가 나오는가?

세 가지 모두 "예"라면 스킬이 정상 동작하는 것입니다.

### Step 5. 문제 발견 시 수정

만약 의도대로 동작하지 않으면 Claude에게 수정을 요청합니다.

```
my-session-wrap 스킬의 [문제점]을 수정해줘
```

`[문제점]` 부분에 구체적으로 무엇이 잘못되었는지 적어줍니다. 예를 들어:

- "에이전트가 2개만 실행돼" → 나머지 2개의 Task 호출이 빠진 것
- "병렬이 아니라 하나씩 실행돼" → Task 호출을 한 메시지에 모아야 함
- "선택지가 안 나와" → AskUserQuestion 호출 부분을 확인

> **실행이 전혀 안 되면 Plan B**: "session-wrap 원본 스킬을 참고해서 내 my-session-wrap을 완성해줘"라고 입력하세요. Claude가 원본을 참고해서 자동 보정합니다.

### Step 6. 교육 세션으로 돌아오기

실습이 끝났으면 **이 교육 세션이 열려 있는 터미널 탭**으로 돌아옵니다. 작업 세션 탭은 열어둔 채로 두어도 괜찮습니다.

> Block 3, 4에서도 같은 방식으로 별도 탭에서 실습하고, 이 교육 세션으로 돌아오면 됩니다.

**정답: 1번. SKILL.md의 frontmatter (name, description)**

스킬이 인식되지 않으면 아무것도 실행되지 않습니다. frontmatter의 `name`이 스킬 호출 이름이 되고, `description`이 자동 매칭의 기준이 됩니다. 기본이 맞아야 나머지가 동작합니다. 에이전트 프롬프트나 Git 상태는 그 다음 문제입니다.
